# MAX30102 心率血氧功能实现说明

本文档整理当前工程中 MAX30102 的接线、初始化、数据读取、心率计算、血氧计算、UI 显示和调试过程。当前代码主要分布在：

- `Core/bsp/mpu6050.c`：MAX30102 的寄存器读写、初始化、FIFO 数据读取。
- `Core/bsp/mpu6050.h`：MAX30102 相关函数声明和调试变量声明。
- `Core/bsp/max30102_reg.h`：MAX30102 寄存器地址定义。
- `Core/bsp/menu.c`：MAX30102 页面 UI、心率血氧算法、传感器任务调用。

## 1. 硬件和通信方式

MAX30102 使用 I2C 通信，当前工程已经改用 `I2C2` 硬件 I2C，和 MPU6050 共用同一组 I2C 总线。

MAX30102 的 7 位地址是 `0x57`，HAL 库使用左移一位后的 8 位地址：

```c
#define MAX30102_ADDRESS 0xAE
```

也就是：

```c
0x57 << 1 = 0xAE
```

当前没有使用 MAX30102 的 `INT` 引脚。这样也能正常采集，只是 MCU 需要定时轮询 FIFO。`INT` 的作用是提醒 MCU “FIFO 有新数据了”，它可以减少轮询和漏读概率，但不会直接提高 SpO2 算法准确度。

## 2. 寄存器定义

寄存器定义在 `Core/bsp/max30102_reg.h`。

当前主要用到：

```c
#define MAX30102_REG_ID            0xFF
#define MAX30102_REG_FIFO_WR_PTR   0x04
#define MAX30102_REG_FIFO_OF_CN    0x05
#define MAX30102_REG_FIFO_RD_PTR   0x06
#define MAX30102_REG_FIFO_DATA     0x07
#define MAX30102_REG_FIFO_CONFIG   0x08
#define MAX30102_REG_MODE_CONFIG   0x09
#define MAX30102_REG_SPO2_CONFIG   0x0A
#define MAX30102_REG_RED_PA        0x0C
#define MAX30102_REG_IR_PA         0x0D
```

注意：`RED_PA` 是红光 LED 电流寄存器，`IR_PA` 是红外 LED 电流寄存器。SpO2 模式下 FIFO 每个样本包含 6 字节，前 3 字节是 RED，后 3 字节是 IR。

## 3. MAX30102 初始化流程

初始化函数在 `Core/bsp/mpu6050.c`：

```c
void max30102_init(void)
```

当前流程：

1. 写 `MODE_CONFIG = 0x40`，复位 MAX30102。
2. 等待复位位自动清零。
3. 读取中断状态寄存器 `0x00` 和 `0x01`，清除旧中断标志。
4. 调用 `max30102_clear_fifo()` 清空 FIFO 指针。
5. 配置 FIFO：

```c
Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_FIFO_CONFIG, 0x4F);
```

6. 配置 SpO2 参数：

```c
Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_SPO2_CONFIG, 0x27);
```

当前含义是使用 100sps、18bit LED 脉宽，作为验证数据链路的配置。

7. 配置 LED 电流：

```c
Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_RED_PA, 0x4F);
Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_IR_PA, 0x1F);
```

这组值是调试过程中为了把 `R` 值从偏低区间拉回正常区间而设置的。之前试过：

```c
RED = 0x24, IR = 0x24
RED = 0x1F, IR = 0x3F
RED = 0x0F, IR = 0x7F
RED = 0x3F, IR = 0x1F
```

这些值会明显影响 RED/IR 的交流直流比例，最终影响 `R` 值和 SpO2 显示。

8. 进入 SpO2 模式：

```c
Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_MODE_CONFIG, 0x03);
```

## 4. FIFO 清空和停止采集

清空 FIFO：

```c
void max30102_clear_fifo(void)
{
    Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_FIFO_WR_PTR, 0x00);
    Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_FIFO_OF_CN, 0x00);
    Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_FIFO_RD_PTR, 0x00);
}
```

退出页面后停止 MAX30102：

```c
void max30102_stop(void)
{
    Sensor_WriteReg(MAX30102_ADDRESS, MAX30102_REG_MODE_CONFIG, 0x80);
}
```

这样可以避免切出页面后传感器还一直采样，导致 FIFO 状态混乱或后台占用 I2C。

## 5. 数据读取流程

读取函数在 `Core/bsp/mpu6050.c`：

```c
void max30102_GetValue(void)
```

流程：

1. 读取中断状态、模式寄存器、FIFO 写指针和读指针。
2. 计算 FIFO 中有多少个样本：

```c
num = (max30102_wr_ptr - max30102_rd_ptr) & 0x1F;
```

3. 如果 `num == 0`，说明 FIFO 暂时没有新数据，直接返回。
4. 如果有数据，从 FIFO DATA 读取 6 字节：

```c
HAL_I2C_Mem_Read(&hi2c2, MAX30102_ADDRESS, MAX30102_REG_FIFO_DATA,
                 I2C_MEMADD_SIZE_8BIT, buf, 6, 100);
```

5. 合成 RED 和 IR 18 位数据：

```c
red = ((uint32_t)buf[0] << 16) | ((uint32_t)buf[1] << 8) | buf[2];
red &= 0x03FFFF;

ir = ((uint32_t)buf[3] << 16) | ((uint32_t)buf[4] << 8) | buf[5];
ir &= 0x03FFFF;
```

6. 设置采样有效标志：

```c
max30102_sample_valid = 1;
```

当前每次只读取 FIFO 中的一个样本。如果主循环卡顿、FIFO 积累很多数据，后续可以改成循环读取多个样本。

## 6. Sensor_Task 调用方式

MAX30102 的周期采样在 `Core/bsp/menu.c` 的 `Sensor_Task()` 中：

```c
if(max30102_flag){
    if (HAL_GetTick() - max_last_tick >= 10) {
        max_last_tick = HAL_GetTick();
        max30102_GetValue();
        max30102_data_proc();
    }
}
```

也就是进入 MAX30102 页面后，每 10ms 采样和处理一次，理论采样频率约 100Hz。

`max30102_flag` 在 MAX30102 页面进入时置 1，退出页面时清 0。

## 7. 页面进入和退出逻辑

MAX30102 页面函数：

```c
void max30102_ui(void)
```

首次进入页面时：

```c
max30102_init();
max30102_algorithm_reset();
max30102_flag = 1;
enter_flag = 1;
```

退出页面时：

```c
page_current = PAGE_MENU;
max30102_flag = 0;
enter_flag = 0;
max30102_stop();
```

这样保证每次进入页面都会重新初始化传感器和算法状态。

## 8. 手指检测

当前用 IR 直流值判断是否放手指：

```c
#define MAX30102_FINGER_TH 50000
```

在 `max30102_data_proc()` 中：

```c
if (ir < MAX30102_FINGER_TH)
{
    max30102_algorithm_reset();
    return;
}
```

如果 IR 低于阈值，认为没有手指或接触不好，清空算法状态，UI 回到 `Put finger`。

如果 IR 高于阈值：

```c
max30102_finger_detected = 1;
```

UI 允许显示 HR、SpO2 和 R 值。

## 9. 心率测量原理

MAX30102 得到的是光强反射值。手指中血液随心跳周期性变化，反射光也会出现周期性波动。

当前算法分两部分：

1. 用低通滤波估计 IR 的直流分量：

```c
if (ir_dc == 0) ir_dc = (int32_t)ir;
ir_dc = (ir_dc * 95 + (int32_t)ir * 5) / 100;
```

2. 用原始 IR 减去直流分量得到交流分量：

```c
ir_ac = (int32_t)ir - ir_dc;
```

`ir_dc` 表示手指、皮肤、环境光等形成的慢变化基线；`ir_ac` 表示心跳造成的快速波动。

然后用 `ir_ac` 的过零上升沿判断一次心跳：

```c
if (ir_ac_prev <= 0 && ir_ac > 0 && ir_ac_abs_avg > HR_MIN_AC_AVG)
```

判断到一次心跳后，计算本次心跳和上一次心跳之间的时间间隔：

```c
interval = now - last_beat_tick;
new_hr = 60000 / interval;
```

例如间隔 800ms：

```c
60000 / 800 = 75 BPM
```

当前保留 45-160 BPM 的结果：

```c
if (new_hr >= 45 && new_hr <= 160)
```

并做平滑：

```c
heart_rate = (heart_rate * 7 + new_hr) / 8;
```

这样可以减少 HR 跳动，但会让响应稍慢。

## 10. SpO2 测量原理

MAX30102 用红光 RED 和红外 IR 估算血氧。基本思想是：含氧血红蛋白和脱氧血红蛋白对红光、红外光的吸收不同，所以 RED 和 IR 的交流/直流比例可以反映血氧变化。

当前先在一个窗口内保存 RED 和 IR 数据：

```c
#define MAX30102_BUF_SIZE 150
```

100Hz 采样下，150 点约为 1.5 秒窗口。

窗口满后计算：

1. RED 和 IR 的直流分量：

```c
red_dc = red_sum / MAX30102_BUF_SIZE;
ir_dc_avg = ir_sum / MAX30102_BUF_SIZE;
```

2. RED 和 IR 的交流分量。

当前不用 `max - min`，而用平均绝对偏差：

```c
red_ac = red_ac_sum / MAX30102_BUF_SIZE;
ir_ac_avg = ir_ac_sum / MAX30102_BUF_SIZE;
```

这样比 `max-min` 更不容易被单个毛刺影响。

3. 计算 R 值：

```c
r = ((float)red_ac / (float)red_dc) /
    ((float)ir_ac_avg / (float)ir_dc_avg);
```

也就是：

```text
R = (RED_AC / RED_DC) / (IR_AC / IR_DC)
```

为了方便 OLED 显示，代码中显示的是：

```c
max30102_r_x100 = (uint16_t)(r * 100.0f + 0.5f);
```

所以屏幕上 `R:100` 表示实际 `R = 1.00`。

4. 根据 R 值估算 SpO2。

当前使用工程标定公式：

```c
spo2_f = 110.0f - 13.0f * r;
```

然后限制显示范围：

```c
if (spo2_f > 100.0f) spo2_f = 100.0f;
if (spo2_f < 70.0f) spo2_f = 70.0f;
```

这个公式不是医疗级公式，是根据当前模块的 R 值区间临时调出来的工程映射。

## 11. 为什么 R 值重要

SpO2 的最终结果主要由 R 值决定。

当前公式下：

```text
R = 0.40 -> SpO2 = 104.8 -> 显示 100
R = 0.80 -> SpO2 = 99.6  -> 显示约 100
R = 1.00 -> SpO2 = 97
R = 1.20 -> SpO2 = 94.4
R = 1.40 -> SpO2 = 91.8
```

所以如果 `R` 长期只有 `40-50`，SpO2 会一直顶到 `99-100`。这不代表真实血氧一定是 100，只说明当前 RED/IR 比例和公式让结果顶到上限。

如果 `R` 长期大于 `140`，SpO2 会偏低，甚至被压到 70 附近。

比较理想的调试目标是：同一根手指、稳定按压时，`R` 大致落在 `80-120` 范围内，再用真实血氧仪对比标定公式。

## 12. LED 电流对 R 值的影响

LED 电流直接影响 RED 和 IR 的直流、交流分量。

经验上：

- 提高 RED 电流，通常会提高 RED 分量，可能让 R 变大。
- 提高 IR 电流，通常会提高 IR 分量，可能让 R 变小。
- 电流太大可能接近饱和，数据看起来稳定，但 AC/DC 比例会失真。
- 电流太小可能 AC 太弱，HR 和 SpO2 都不稳定。

当前代码使用：

```c
RED = 0x4F;
IR  = 0x1F;
```

这是为了把之前偏低的 `R=40-50` 往上拉。烧录后应重点观察 OLED 上的 `R:xxx`。

## 13. 当前 UI 显示内容

MAX30102 页面显示：

```text
MAX30102
HR  :xxx BPM
SpO2:xxx %
R:xxx
```

如果未检测到手指：

```text
Put finger
Waiting...
```

其中：

- `HR` 是心率，单位 BPM。
- `SpO2` 是估算血氧。
- `R` 是调试值，显示 `R * 100`。

`R` 是当前调试中最重要的判断依据。

## 14. 当前算法中的等待和滤波

当前参数：

```c
#define MAX30102_WARMUP_SAMPLE 50
#define MAX30102_HR_VALID_BEATS 1
#define SPO2_SKIP_WINDOWS 3
```

含义：

- `MAX30102_WARMUP_SAMPLE = 50`：刚放手指后先等待约 0.5 秒，不用于 HR 计算。
- `MAX30102_HR_VALID_BEATS = 1`：检测到 1 次有效心跳间隔后允许显示 HR。
- `SPO2_SKIP_WINDOWS = 3`：刚放手指后的前 3 个 SpO2 窗口不显示，避免初始低值进入平滑，造成从 70 慢慢爬升一分钟的现象。

SpO2 平滑：

```c
if (spo2_valid)
    spo2 = (spo2 * 3 + new_spo2) / 4;
else
    spo2 = new_spo2;
```

第一次有效值直接显示，后续用 3/4 平滑。

## 15. 当前实现的局限

当前 MAX30102 功能可以用于课程项目、演示和趋势观察，但不能作为准确医疗测量。

主要原因：

1. 没有使用标准血氧仪做多点标定。
2. SpO2 公式是工程调试公式，不是针对当前模块、手指、光路校准后的公式。
3. 反射式测量对手指压力、环境光、模块贴合程度非常敏感。
4. 面包板和杜邦线连接容易引入接触不良、供电波动和 I2C 干扰。
5. 当前 FIFO 每次只读取一个样本，主循环卡顿时可能积压数据。

## 16. 建议的后续优化

1. 使用商品血氧仪做对比，记录多组数据：

```text
R值    商品血氧仪SpO2    本项目SpO2
```

然后重新拟合公式。

2. OLED 临时显示更多调试量：

```text
RED_AC
IR_AC
RED_DC
IR_DC
```

这样可以判断是光强太低、太高，还是通道比例不对。

3. 增加 INT 引脚。

INT 不会直接提高 SpO2 精度，但可以在 FIFO 有新数据时立即读取，减少轮询和漏读。

4. FIFO 一次读取多个样本。

当前 `num > 0` 时只读一个样本。后续可以根据 `num` 循环读取 FIFO，避免主循环卡顿时数据滞后。

5. 改进 HR 检测。

当前用过零上升沿。后续可以改成峰值检测、自适应阈值、移动平均或成熟的心率算法。

6. 改善硬件。

建议 MAX30102 供电稳定、GND 可靠、I2C 上拉合适，传感器与手指之间减少漏光和晃动。

## 17. 调试判断表

| 现象 | 可能原因 | 处理方向 |
| --- | --- | --- |
| ID 读不到 | 地址错误、接线错误、供电错误 | 检查 `0xAE`、VIN/GND/SCL/SDA |
| IR/RED 都是 0 | FIFO 没数据、模式没启动、I2C 读失败 | 检查 `MODE_CONFIG=0x03` 和 FIFO 指针 |
| 一直 Put finger | `ir < 50000` | 看 IR 原始值，调整手指位置或阈值 |
| HR 乱跳 | 手指晃动、AC 太弱、阈值不合适 | 稳定按压，调整 `HR_MIN_AC_AVG` |
| SpO2 一直 70 | R 太大或公式不合适 | 降 RED 或升 IR，观察 R |
| SpO2 一直 100 | R 太小或公式顶满 | 升 RED 或降 IR，观察 R |
| SpO2 从 70 慢慢爬升 | 初始低值进入平滑 | 跳过前几个窗口，减弱平滑 |

## 18. 当前最重要的测试方法

烧录后不要只看 SpO2，先看 `R:xxx`。

测试步骤：

1. 进入 MAX30102 页面。
2. 手指稳定覆盖传感器，尽量不要漏光。
3. 等 10-20 秒。
4. 记录：

```text
HR =
SpO2 =
R =
```

5. 多测试几次，看 R 是否稳定。

如果 R 能稳定在 `80-120`，再讨论 SpO2 公式是否需要微调。如果 R 长期偏离这个范围，优先调 LED 电流和手指贴合方式，不要直接改 SpO2 公式。
